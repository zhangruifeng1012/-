## 通用约定

### 用例编号规则

`TC-<模块缩写>-<序号>`，模块缩写：LOGIN(登录)、CRAW(采集)、LIST(列表)、DEDUP(去重)、ANALYSIS(分析)、PARSE(解析)、AUTH(权限)、UI(界面)、TASK(任务)

### 优先级定义

| 标记 | 优先级 | 含义 |
|------|-------|------|
| P0 | 高 | 核心功能，必须 100% 通过 |
| P1 | 中 | 重要功能，通过率 ≥ 95% |
| P2 | 低 | 边界/兼容性，尽量通过 |

### 执行结果填写规范

| 结果 | 含义 |
|------|------|
| PASS | 通过 |
| FAIL | 失败（需提 Bug） |
| BLOCK | 阻塞（环境/依赖问题） |
| N/A | 不适用 |

### 测试前置条件（全局）

1. 已执行 `pip install -r requirements.txt` 安装依赖
2. 已执行 `python init_data.py` 生成 150 条示例数据
3. 服务已通过 `python app.py` 启动，监听 `http://127.0.0.1:5000`
4. 测试账号：tester / test123（如不存在，首次登录会自动注册）
