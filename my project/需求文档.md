## 一、产品概述

### 1.1 产品背景

政府及企业招标信息分散在多个公开平台（如中国政府采购网、中国招标投标公共服务平台等），人工检索效率低、覆盖面有限。本项目旨在构建一套**智能招标信息采集系统**（品牌名：智招云），通过爬虫技术自动采集、去重、存储招标数据，并利用 ECharts 可视化进行多维分析，帮助用户快速掌握招标市场动态。

### 1.2 产品目标

| 目标 | 描述 |
|------|------|
| 自动采集 | 支持按关键词 / URL 两种模式采集招标数据 |
| 智能去重 | 基于 Redis 集合 + 数据库唯一索引的双重去重机制，避免重复入库 |
| 多维筛选 | 按地区、行业、关键词、日期范围、排序方式组合检索 |
| 趋势分析 | 提供发布趋势、地区分布、行业分布、高频关键词、热力图等 5 种可视化图表 |
| 系统稳定 | Session 认证、Redis 缓存、结构化异常处理 |

### 1.3 目标用户

- 政府采购从业人员
- 招标代理机构工作人员
- 企业投标/商务拓展人员
- 行业数据分析师

## 二、功能需求

### 2.1 用户认证模块

**FR-AUTH-01 用户登录/自动注册**

| 属性 | 描述 |
|------|------|
| 优先级 | P0（最高） |
| 描述 | 用户输入用户名和密码，系统验证身份。若用户不存在则自动创建（首次登录即注册） |
| 前置条件 | 无 |
| 基本流程 | 1. 用户访问登录页 → 2. 输入用户名/密码 → 3. 点击"登录/注册" → 4. 系统验证 → 5. 登录成功跳转控制台 |
| 异常流程 | 用户名/密码为空或全空格 → 提示"用户名和密码不能为空"；已注册用户密码错误 → 提示"密码错误" |
| 验收标准 | 新用户首次登录提示"注册并登录成功"；老用户提示"登录成功"；成功后携带 Session Cookie |

**FR-AUTH-02 用户登出**

| 属性 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 清除用户 Session，重定向至登录页 |
| 验收标准 | 登出后访问任何需认证页面均被重定向到登录页 |

**FR-AUTH-03 访问控制**

| 属性 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 未登录用户访问 API 返回 401 JSON；访问页面路由返回 302 重定向至登录页 |
| 验收标准 | 所有 `/api/*` 接口和 `/dashboard`、`/list`、`/analysis`、`/detail/*` 页面均受保护 |

### 2.2 数据采集模块

**FR-CRAW-01 关键词采集**

| 属性 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 用户输入关键词（中英文皆可，上限 100 字符），系统按关键词生成模拟招标数据并入库 |
| 输入 | 关键词（string，1-100 字符） |
| 输出 | 成功采集条数 count、新增数据列表 data（最多返回前 5 条预览） |
| 异常流程 | 关键词为空 → "请输入关键词或URL"；关键词超 100 字符 → "关键词过长" |
| 验收标准 | 采集成功后：①数据入库 bidding_data 表；②crawl_tasks 表记录任务；③统计面板 total_records 增长 |

**FR-CRAW-02 URL 采集**

| 属性 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 用户输入目标 URL，系统验证格式后采集数据 |
| 输入 | URL（string，需 http/https/ftp 协议） |
| 输出 | 采集结果同 FR-CRAW-01，额外支持 deduped 标识 |
| 异常流程 | URL 格式无效 → "URL格式无效，请检查" |
| 验收标准 | 重复 URL 第二次采集返回 deduped=True，count=0 |

**FR-CRAW-03 优先级规则**

| 属性 | 描述 |
|------|------|
| 优先级 | P1 |
| 描述 | 当关键词和 URL 同时提供时，按 URL 模式执行 |
| 验收标准 | 同时填写两者，系统以 URL 优先采集 |

**FR-CRAW-04 采集任务记录**

| 属性 | 描述 |
|------|------|
| 优先级 | P1 |
| 描述 | 每次采集操作自动记录到 crawl_tasks 表，包含用户 ID、关键词、URL、状态、结果数量、创建时间 |
| 验收标准 | 采集后可在控制台"最近采集任务"列表中查看 |

### 2.3 数据去重模块

**FR-DEDUP-01 双重去重机制**

| 属性 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 采用 Redis URL 集合（MD5 哈希）+ 数据库 url_hash 唯一索引两层去重 |
| 去重流程 | ①计算 URL 的 MD5 哈希 → ②查询 Redis 集合是否已存在 → ③查询数据库 url_exists → ④均不命中才插入 |
| 验收标准 | 相同 URL 不会重复入库；去重测试接口 first_insert=True, second_insert=False |

**FR-DEDUP-02 缓存统计**

| 属性 | 描述 |
|------|------|
| 优先级 | P1 |
| 描述 | 统计面板展示已缓存的去重 URL 数量 |
| 验收标准 | 采集新数据后 cached_urls 数值增加 |

### 2.4 招标列表与筛选模块

**FR-LIST-01 列表查询**

| 属性 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 以表格形式展示招标数据，支持分页（默认每页 20 条） |
| 显示字段 | 序号、标题（可点击进详情）、地区、行业、发布日期、预算（万元）、来源、操作 |
| 验收标准 | 列表正确渲染；分页功能正常；翻页数据不重复不遗漏 |

**FR-LIST-02 多维筛选**

| 属性 | 描述 |
|------|------|
| 优先级 | P0 |
| 描述 | 支持地区（10 个枚举值含"全部"）、行业（7 个枚举值）、关键词模糊搜索（匹配 title 和 keywords 字段）、日期范围、排序方式（按日期/按预算）组合筛选 |
| 验收标准 | 各筛选条件独立生效；组合筛选取交集；无匹配结果时显示"暂无数据" |

**FR-LIST-03 查询缓存**

| 属性 | 描述 |
|------|------|
| 优先级 | P1 |

| 验收标准 | 首次请求 from_cache=False；相同参数再次请求 from_cache=True |

## 三、非功能需求

### 3.1 性能需求

| 编号 | 需求 | 指标 |
|------|------|------|
| NFR-PERF-01 | 列表查询响应时间 | 首次请求 ≤ 2 秒；缓存命中 ≤ 50ms |
| NFR-PERF-02 | 采集接口响应时间 | 关键词采集 ≤ 5 秒；URL 采集 ≤ 10 秒 |
| NFR-PERF-03 | 缓存命中率 | 同一参数重复请求场景 ≥ 80% |

### 3.2 安全需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-SEC-01 | Session 认证 | 基于 Flask Session Cookie，未登录访问返回 401/302 |
| NFR-SEC-02 | 密码存储 | MD5 哈希存储（生产环境建议升级为 bcrypt） |
| NFR-SEC-03 | SQL 注入防护 | 使用参数化查询（sqlite3 ? 占位符） |
| NFR-SEC-04 | 输入校验 | 关键词长度限制 100 字符；URL 格式正则校验 |

### 3.3 兼容性需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-COMPAT-01 | 浏览器 | Chrome 80+、Edge 80+ |
| NFR-COMPAT-02 | 分辨率 | 1920×1080（主要）、1366×768（兼容） |
| NFR-COMPAT-03 | 响应式 | ≤768px 时侧栏收窄为图标模式 |

### 3.4 数据需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-DATA-01 | 数据库 | SQLite（data/bidding.db），4 张表：users、bidding_data、user_subscriptions、crawl_tasks |
| NFR-DATA-02 | 缓存 | 基于 JSON 文件的 Redis 模拟（data/cache.json），支持 KV 缓存和 Set 集合 |
| NFR-DATA-03 | 初始数据 | init_data.py 生成 150 条示例数据，覆盖 9 地区 × 7 行业 |

## 四、系统架构概述

### 4.1 技术栈

| 层次 | 技术 |
|------|------|
| 后端框架 | Flask 3.0.0 |
| 数据库 | SQLite 3 |
| 缓存 | JSON 文件模拟 Redis（支持 get/set/sadd/sismember + TTL） |
| 爬虫 | requests 2.31.0 + BeautifulSoup4 4.12.2 + lxml 4.9.3 |
| 前端 | 原生 HTML/CSS/JavaScript |
| 图表 | ECharts 5.4.3（CDN）+ echarts-wordcloud 2.1.0（CDN） |
| 模板引擎 | Jinja2 3.1.2 |

### 4.2 页面结构

| 页面 | 路由 | 功能 |
|------|------|------|
| 登录页 | GET /login | 用户认证入口 |
| 控制台 | GET /dashboard | 统计面板 + 采集表单 + 任务记录 + 测试工具 |
| 招标列表 | GET /list | 多维筛选 + 表格 + 分页 |
| 趋势分析 | GET /analysis | 5 个 ECharts 可视化图表 |
| 招标详情 | GET /detail/{id} | 单条记录完整信息 |

### 4.3 数据模型

**bidding_data 表（核心）**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTO | 自增主键 |
| title | TEXT | NOT NULL | 标题 |
| content | TEXT | - | 内容摘要 |
| url_hash | TEXT | UNIQUE | URL 的 MD5 哈希，用于去重 |
| url | TEXT | - | 原始链接 |
| region | TEXT | - | 地区 ID |
| industry | TEXT | - | 行业 ID |
| publish_date | TEXT | - | 发布日期 |
| source | TEXT | - | 来源网站 |
| budget | REAL | - | 预算（万元） |
| keywords | TEXT | - | 关键词（逗号分隔） |
| created_at | TEXT | - | 入库时间 |

**users 表**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTO | 自增主键 |
| username | TEXT | UNIQUE NOT NULL | 用户名 |
| password | TEXT | NOT NULL | MD5 哈希密码 |
| email | TEXT | - | 邮箱（自动注册时为占位值） |
| created_at | TEXT | - | 注册时间 |

**crawl_tasks 表**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTO | 自增主键 |
| user_id | INTEGER | - | 关联用户 |
| keyword | TEXT | - | 采集关键词 |
| url | TEXT | - | 采集 URL |
| status | TEXT | - | completed / duplicate / failed |
| result_count | INTEGER | - | 本次新入库条数 |
| created_at | TEXT | - | 创建时间 |

## 五、已知限制与风险

| 编号 | 限制/风险 | 影响 | 建议 |
|------|----------|------|------|
| LIM-01 | 爬虫数据为模拟生成 | 非"真实"网页抓取 | 上线前需替换为真实爬虫逻辑 |
| LIM-02 | 缓存为 JSON 文件模拟 | 非真正 Redis，高并发性能受限 | 生产环境建议迁移至真实 Redis |
| LIM-03 | 密码使用 MD5 存储 | 安全性不足 | 建议升级为 bcrypt/scrypt |
| LIM-04 | 热力图数据为前端随机生成 | 与真实数据库数据不关联 | 需后端提供对应 API |
| LIM-05 | 无用户权限管理 | 所有用户权限相同 | 后续可增加角色系统 |
| LIM-06 | 无定时任务 | 采集需手动触发 | 可集成 Celery/APScheduler |
