## 方式一：传统服务器部署（推荐）

### 1. 上传代码到服务器

```bash
# 方式 A：使用 Git（推荐）
cd /opt
git clone <你的Git仓库地址> zhaobiao
cd zhaobiao

# 方式 B：使用 SCP / SFTP 上传
# 在本地执行（将整个项目目录压缩后上传）
# tar -czf zhaobiao.tar.gz my-project/
# scp zhaobiao.tar.gz user@server:/opt/
# 然后在服务器解压：
# cd /opt && tar -xzf zhaobiao.tar.gz
```

### 2. 创建虚拟环境并安装依赖

```bash
cd /opt/zhaobiao

# 创建虚拟环境
python3 -m venv venv
# 或使用 conda（如果已安装）
# conda create -n zhaobiao python=3.10
# conda activate zhaobiao

# 激活虚拟环境
source venv/bin/activate

# 安装依赖
pip install --upgrade pip
pip install -r requirements.txt

# 验证依赖
python -c "import flask; print('Flask:', flask.__version__)"
```

### 3. 初始化数据库和示例数据

```bash
# 生成示例数据（首次部署必执行）
python init_data.py

# 验证：查看数据库文件
ls -la data/
# 预期输出：bidding.db, cache.json
```

### 4. 测试启动（开发模式）

```bash
# 方式 A：直接运行（前台）
python app.py
# 访问 http://服务器IP:5000 测试

# 方式 B：指定端口和地址
python app.py --host 0.0.0.0 --port 8080
```

### 5. 配置 Systemd 守护进程（生产环境必需）

创建 `/etc/systemd/system/zhaobiao.service`：

```ini
[Unit]
Description=Smart Bidding Crawler System
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/zhaobiao
Environment="PATH=/opt/zhaobiao/venv/bin"
ExecStart=/opt/zhaobiao/venv/bin/python /opt/zhaobiao/app.py
Restart=always
RestartSec=5

# 日志输出
StandardOutput=append:/var/log/zhaobiao.log
StandardError=append:/var/log/zhaobiao-error.log

[Install]
WantedBy=multi-user.target
```

启动并设置开机自启：

```bash
# 重新加载 systemd 配置
systemctl daemon-reload

# 启动服务
systemctl start zhaobiao

# 查看状态
systemctl status zhaobiao

# 设置开机自启
systemctl enable zhaobiao

# 查看日志
journalctl -u zhaobiao -f
# 或查看日志文件
tail -f /var/log/zhaobiao.log
```

### 6. 开放防火墙端口

```bash
# CentOS / RHEL
firewall-cmd --permanent --add-port=5000/tcp
firewall-cmd --reload
firewall-cmd --list-ports

# Ubuntu / Debian
ufw allow 5000/tcp
ufw status

# 验证端口监听
netstat -tlnp | grep 5000
# 或
ss -tlnp | grep 5000
```

## 方式二：Docker 容器部署

### 1. 创建 `Dockerfile`

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 设置环境变量
ENV PYTHONDONTWRITEBYTECODE 1
ENV PYTHONUNBUFFERED 1

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制项目文件
COPY . .

# 暴露端口
EXPOSE 5000

# 启动命令
CMD ["python", "app.py"]
```

### 2. 创建 `docker-compose.yml`（可选，便于管理）

```yaml
version: '3.8'

services:
  zhaobiao:
    build: .
    container_name: zhaobiao-app
    ports:
      - "5000:5000"
    volumes:
      - ./data:/app/data
    restart: unless-stopped
    environment:
      - TZ=Asia/Shanghai
```

### 3. 构建并启动容器

```bash
# 构建镜像
docker build -t zhaobiao:v1.0 .

# 首次运行需初始化数据（临时运行容器生成数据）
docker run --rm -v $(pwd)/data:/app/data zhaobiao:v1.0 python init_data.py

# 使用 Docker Compose 启动（推荐）
docker-compose up -d

# 或直接使用 Docker 命令
docker run -d --name zhaobiao-app \
  -p 5000:5000 \
  -v $(pwd)/data:/app/data \
  --restart unless-stopped \
  zhaobiao:v1.0

# 查看运行状态
docker ps
docker logs zhaobiao-app

# 停止服务
docker-compose down
# 或
docker stop zhaobiao-app
```

## 方式三：Nginx + uWSGI 生产部署

### 1. 安装 uWSGI

```bash
source venv/bin/activate
pip install uwsgi
```

### 2. 创建 uWSGI 配置文件 `zhaobiao.ini`

```ini
[uwsgi]
master = true
processes = 4
threads = 2
socket = 127.0.0.1:8000
chdir = /opt/zhaobiao
module = app:app
home = /opt/zhaobiao/venv
virtualenv = /opt/zhaobiao/venv
wsgi-file = app.py
callable = app
logto = /var/log/uwsgi/zhaobiao.log
chmod-socket = 660
vacuum = true
die-on-term = true
pidfile = /var/run/zhaobiao.pid
```

### 3. 启动 uWSGI

```bash
# 创建日志目录
mkdir -p /var/log/uwsgi

# 测试运行
uwsgi --ini zhaobiao.ini

# 使用 Systemd 管理（创建 /etc/systemd/system/zhaobiao-uwsgi.service）
```

### 4. 配置 Nginx 反向代理

创建 `/etc/nginx/conf.d/zhaobiao.conf`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 10M;

    location / {
        include uwsgi_params;
        uwsgi_pass 127.0.0.1:8000;

        # 超时设置
        uwsgi_connect_timeout 300;
        uwsgi_send_timeout 300;
        uwsgi_read_timeout 300;

        # Header 传递
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态文件缓存（可选，提升性能）
    location /static/ {
        alias /opt/zhaobiao/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 访问日志
    access_log /var/log/nginx/zhaobiao-access.log;
    error_log /var/log/nginx/zhaobiao-error.log;
}
```

### 5. 测试并重启 Nginx

```bash
# 测试配置
nginx -t

# 重启
systemctl restart nginx
# 或
nginx -s reload

# 查看状态
systemctl status nginx
```

### 6. 配置 HTTPS（推荐，使用 Let's Encrypt）

```bash
# 安装 certbot
apt-get install certbot python3-certbot-nginx  # Ubuntu/Debian
# 或
yum install certbot python3-certbot-nginx      # CentOS

# 自动获取并配置证书
certbot --nginx -d your-domain.com

# 证书会自动续期，也可手动测试
certbot renew --dry-run
```

## 常用运维命令

```bash
# 启动服务
systemctl start zhaobiao

# 停止服务
systemctl stop zhaobiao

# 重启服务
systemctl restart zhaobiao

# 查看运行状态
systemctl status zhaobiao

# 查看实时日志
journalctl -u zhaobiao -f --since "today"

# 查看数据目录
ls -la /opt/zhaobiao/data/

# 备份数据库
cp /opt/zhaobiao/data/bidding.db /backup/bidding-$(date +%Y%m%d).db

# 手动触发数据采集
cd /opt/zhaobiao
source venv/bin/activate
python -c "from crawler import crawler; data = crawler.crawl_by_keyword('软件开发'); print(f'采集到 {len(data)} 条')"
```

### 定时任务示例（每天自动采集）

编辑 crontab：
```bash
crontab -e
```

添加以下内容：
```cron
# 每天凌晨 2:00 采集新数据
0 2 * * * cd /opt/zhaobiao && /opt/zhaobiao/venv/bin/python -c "from crawler import crawler; crawler.crawl_by_keyword('软件开发')" >> /var/log/zhaobiao-cron.log 2>&1

# 每天凌晨 3:00 备份数据库
0 3 * * * cp /opt/zhaobiao/data/bidding.db /backup/bidding-$(date +\%Y\%m\%d).db

# 每周日凌晨 4:00 清理 30 天前的备份
0 4 * * 0 find /backup/ -name "bidding-*.db" -mtime +30 -delete
```

## 常见问题排查

### ❌ 问题 1：端口 5000 被占用

```bash
# 查看占用端口的进程
lsof -i :5000
netstat -tlnp | grep 5000

# 杀掉进程
kill -9 <PID>
```

### ❌ 问题 2：Python 模块找不到

```bash
# 确认虚拟环境激活
source venv/bin/activate
which python   # 应显示 /opt/zhaobiao/venv/bin/python

# 重新安装
pip install -r requirements.txt
```

### ❌ 问题 3：无法访问 Web 界面

```bash
# 1. 检查服务是否运行
systemctl status zhaobiao

# 2. 检查端口监听
netstat -tlnp | grep 5000

# 3. 检查防火墙
firewall-cmd --list-ports   # CentOS
ufw status                   # Ubuntu

# 4. 本地测试
curl http://127.0.0.1:5000
```

### ❌ 问题 4：数据库文件权限

```bash
# 设置正确权限
chown -R root:root /opt/zhaobiao/data
chmod -R 755 /opt/zhaobiao/data
chmod 644 /opt/zhaobiao/data/bidding.db
```

### ❌ 问题 5：中文乱码

```bash
# 设置系统字符集
export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8

# 或在 Systemd 配置中添加
Environment=LANG=en_US.UTF-8
```

### ❌ 问题 6：Echarts 图表不显示

- 检查浏览器控制台（F12）是否有 JS 错误
- 确认 CDN 链接可访问（Echarts 使用 CDN 加载）
- 如服务器在内网无外网，需要下载 echarts.min.js 到本地 static/js 目录

```bash
# 下载本地 Echarts
cd /opt/zhaobiao/static/js
wget https://cdn.jsdelivr.net/npm/echarts@5.4.3/dist/echarts.min.js
wget https://cdn.jsdelivr.net/npm/echarts-wordcloud@2.1.0/dist/echarts-wordcloud.min.js

# 然后修改 templates 中的 <script src> 路径为本地路径
```

## 性能优化建议

| 优化项 | 建议配置 |
|-------|---------|
| **Worker 进程数** | CPU 核心数 × 2 |
| **Redis 缓存过期时间** | 查询缓存 300 秒 / URL去重永久 |
| **数据库定期清理** | 每月清理 3 个月前的历史数据 |
| **Nginx Gzip 压缩** | 开启 `gzip on; gzip_types text/css application/javascript application/json;` |
| **静态资源缓存** | 30 天强缓存（见上方 Nginx 配置） |

## 数据备份与恢复

```bash
# ===== 备份 =====
# 方式 A：数据库文件拷贝
cp data/bidding.db backup/bidding-$(date +%Y%m%d).db

# 方式 B：导出为 SQL（便于跨库迁移）
sqlite3 data/bidding.db .dump > backup/bidding-$(date +%Y%m%d).sql

# ===== 恢复 =====
# 恢复方式 A：替换文件
cp backup/bidding-20240101.db data/bidding.db

# 恢复方式 B：从 SQL 导入
sqlite3 data/bidding.db < backup/bidding-20240101.sql
```

## 安全加固建议

1. **修改默认密钥**：编辑 `app.py` 中的 `SECRET_KEY`
2. **启用 HTTPS**：使用 Let's Encrypt 或商业证书
3. **限制访问来源**：在 Nginx 配置中添加 IP 白名单
4. **定期更新依赖**：`pip list --outdated` 检查并更新
5. **数据库访问控制**：限制 data 目录权限
6. **日志监控**：接入 ELK / Prometheus 等监控系统
