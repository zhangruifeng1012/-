# 智能招标信息采集系统 — 测试计划

**文档版本**：v1.0  
**测试范围**：登录/注册、招标列表、自定义爬虫、数据去重、Redis缓存、趋势图表、HTML解析  
**测试环境**：Python 3.10+ / Flask / SQLite / 浏览器 Chrome & Edge / 分辨率 1920×1080 & 1366×768  
**预计总测试用例**：约 **85 条**

---

## 一、测试策略

### 1.1 测试类型

| 类型 | 工具/方式 | 目标覆盖 |
|------|----------|---------|
| 功能测试 | 手工 + Postman | 所有页面核心功能、API接口 |
| 接口测试 | Postman / curl | 9 个 API 接口、请求/响应、缓存一致性 |
| UI 自动化 | Selenium IDE | 登录→采集→查看 核心流程回归 |
| 异常测试 | 手工 + 构造请求 | URL格式、空参数、边界值、去重逻辑 |
| 兼容性测试 | Chrome / Edge | 页面布局、图表渲染、响应式 |
| 性能测试 | 手工观察 / curl 计时 | 列表分页、缓存命中率、爬虫响应 |

### 1.2 测试方法

- **等价类划分**：关键词（有效/无效/中英文/特殊字符）、URL（合法/非法/空/重复）、地区/行业枚举值
- **边界值分析**：关键词长度 0 / 1 / 100 / 101，分页 1 / 0 / 超大值，日期边界
- **场景化测试**：模拟真实用户从登录→采集→筛选→分析全链路
- **反向测试**：未登录访问API、重复提交、SQL注入字符、XSS字符

## 四、缺陷管理

- **严重等级**：阻断(Blocker) > 高(Critical) > 中(Major) > 低(Minor)
- **缺陷模板**：标题 / 复现步骤 / 实际结果 / 预期结果 / 环境信息 / 截图附件
- **回归验证**：Bug 修复后需执行相关 TC，并验证缓存数据一致性

## 五、验收通过标准

1. 所有 ⭐ 高 优先级用例 100% 通过
2. 中低优先级用例通过率 ≥ 95%，且无严重缺陷
3. Chrome & Edge 双浏览器下页面无错位、无 JS 报错
4. Redis 缓存命中率 ≥ 80%（同一参数重复请求场景）
5. URL去重逻辑阻断率 ≥ 99%（相同URL二次采集不会重复入库）
